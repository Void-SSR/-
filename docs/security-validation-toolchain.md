# 安全验证工具链（Web 版）

最后更新：2026-03-17

这份文档描述《拯救美少女》Web 版在引入新工具前所采用的安全验证基线。

## 1. 目标

- 在新工具正式使用前，先做安全验证
- 优先降低来源、完整性、依赖、残留和脏环境风险
- 形成可复用、可追踪、可审计的验证链路

## 2. 正式基线

### 2.1 macOS 原生验证
- `spctl`
- `codesign`
- `pkgutil`
- `shasum`
- `xattr`
- `otool`
- `file`

### 2.2 供应链与扫描
- `cosign`
- `trivy`
- `git-lfs`

## 3. 使用原则

1. 任何新工具正式使用前，先核验来源是否官方或高可信。
2. 再根据工具形态，使用上述基线做完整性、签名、依赖或残留风险检查。
3. 验证通过后，才允许进入正式工具链。
4. 如果当前能力不足，先补齐验证能力，再引入新工具。

## 4. 说明

- Web 版与 Cocos 版共用同一台机器上的安全验证能力，但在记录体系上独立。
- 这份文档服务于 Web 版日志看板和版本记录体系。
