# Web 版错误库

最后更新：2026-03-19

这份错误库用于记录《拯救美少女》Web 版开发中未达成目标的情况、真实卡点、根因、解决方式，以及最终沉淀出的规范。
目标不是追责，而是减少重复犯错，让流程持续进化。

## 记录结构

每条记录至少包含：
- `时间`
- `目标`
- `未达成/卡点`
- `根因`
- `解决方式`
- `沉淀规范`

## 错误记录

### E-001 线上日志板字段未同步
- 时间：2026-03-19
- 目标：把 Web 日志板中的 `当前标签 / 当前提交` 改成 `当前版本标签号 / 当前Git提交号`
- 未达成/卡点：本地文件已改，但线上页面仍显示旧文案
- 根因：只修改了本地文件，没有在说明时明确区分“本地已完成”和“线上未推送”
- 解决方式：
  - 核对改动目标确实是 Web 版日志板
  - 明确说明线上未更新的真实原因
  - 将这类情况纳入网址提供规则
- 沉淀规范：
  - 只有在对应改动已确认成功并上线后，才提供对应网址
  - 必须明确区分 `本地已改` 与 `线上已生效`

### E-002 推送链路多重阻塞
- 时间：2026-03-19
- 目标：把 Web 版里程碑提交推送到 GitHub `main`
- 未达成/卡点：推送先后遇到 DNS 解析失败、SSH 公钥权限失败、`git-lfs` 不在 PATH 等问题
- 根因：
  - 当前环境的推送链路没有一次性验证完整
  - SSH 连接、GitHub 身份与 LFS 运行环境属于不同层级问题，初次处理时没有按层拆开
- 解决方式：
  - 恢复并核对正确远程地址
  - 指定 GitHub 专用 SSH key
  - 补上 `/opt/homebrew/bin`，让 `git-lfs` 在 push 时可用
  - 最终成功推送 `main`
- 沉淀规范：
  - 以后遇到推送失败时，按 `网络解析 -> SSH 身份 -> LFS/依赖环境` 的顺序排查
  - 推送前优先确认远程地址、SSH key、`git-lfs` 三项

### E-003 在结果未成立前提前输出网址
- 时间：2026-03-19
- 目标：在每轮完成后给出对应网址
- 未达成/卡点：曾在改动尚未真正上线时，先给出了网址，导致用户打开后看到旧页面
- 根因：没有把“成功标准”和“网址提供时机”绑定在一起
- 解决方式：
  - 建立独立网址提供规则
  - 要求先确认改动真正成功，再提供对应网址
- 沉淀规范：
  - 改 `Web 版日志板`，只在日志板改动成功后提供日志板网址
  - 改 `Web 版游戏`，只在游戏改动成功后提供游戏网址
  - 需求理解和标准对齐阶段不提供任何结果网址

## 当前长期规范摘要

- 任何未达成目标的情况，如果影响了用户判断或项目推进，都应进入错误库。
- 记录错误时，必须写清根因和最终可复用的规范。
- 错误库与标准库相互联动：成熟规则进入标准库，具体案例保留在错误库。
